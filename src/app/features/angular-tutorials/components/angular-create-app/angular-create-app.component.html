Creating an Angular App

Step-by-step setup guide with best practices

01
Create a new Angular app

Open your terminal and run the Angular CLI command to scaffold a new project.

bash 
ng new my-app
02
Add provideHttpClient() in app.config.ts

Register the HTTP client provider so your services can make API requests throughout the application.

typescript — app.config.ts 
import { ApplicationConfig, provideZoneChangeDetection } from '@angular/core';
import { provideRouter } from '@angular/router';
import { routes } from './app.routes';
import { provideHttpClient } from '@angular/common/http';

export const appConfig: ApplicationConfig = {
  providers: [
    provideHttpClient(),
    provideZoneChangeDetection({ eventCoalescing: true }),
    provideRouter(routes)
  ]
};
03
Create directories under the app folder

Organize your project with three top-level folders inside src/app/ to keep concerns separated and the codebase scalable.

📁 src/app/
├── 📁 core/ new
├── 📁 features/ new
└── 📁 shared/ new
💡 core/ — singleton services, guards, interceptors  |  features/ — feature modules  |  shared/ — reusable components and pipes
04
Create a feature module folder structure

Inside features/, create a folder for each feature (e.g. item) with three subfolders for clean separation.

📁 features/
└── 📁 item/ new
    ├── 📁 components/ new
    ├── 📁 models/ new
    └── 📁 services/ new
💡 components/ — UI components  |  models/ — interfaces & types  |  services/ — API calls & business logic
05
Generate a component using Angular CLI

Use the Angular CLI to generate a component inside the correct feature folder.

bash 
ng generate component features/item/components/item-list

This generates the following files:

📁 features/item/components/item-list/
├── item-list.component.ts new
├── item-list.component.html new
├── item-list.component.css new
└── item-list.component.spec.ts new
