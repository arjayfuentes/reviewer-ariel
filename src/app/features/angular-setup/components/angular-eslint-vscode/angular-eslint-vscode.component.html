ESLint Setup Guide

Angular + VS Code — From Installation to Verification

Part 1 — Installation
01
Install the ESLint Extension in VS Code

Open VS Code and press Ctrl+Shift+X to open the Extensions panel. Search for ESLint and install the one published by Microsoft.

Extension ID 
dbaeumer.vscode-eslint  (Microsoft)

Alternatively, use the Quick Open shortcut. Press Ctrl+P and paste:

VS Code Quick Open 
ext install dbaeumer.vscode-eslint
02
Add ESLint to Your Angular Project

Open a terminal inside your Angular project folder and run:

bash 
ng add @angular-eslint/schematics

This command automatically does the following:

✓ Installs all required ESLint packages
✓ Creates an .eslintrc.json (or eslint.config.js) in your project root
✓ Updates angular.json to wire up the lint target
03
Configure VS Code to Auto-Fix on Save

Press Ctrl+Shift+P, type Open User Settings JSON, and press Enter. Add the following lines inside the JSON object:

settings.json 
"editor.codeActionsOnSave": {
  "source.fixAll.eslint": "explicit"
},
"eslint.validate": [
  "javascript",
  "typescript",
  "html"
]
💡 This tells VS Code to run ESLint automatically every time you save a file.
Part 2 — Testing ESLint Works
M1
Method 1 — Run from the Terminal

In your project terminal, run:

bash 
ng lint

What to expect:

Status Meaning Detail
OK All files pass linting No issues found
! List of errors/warnings File path, line, rule name
💡 Run ng lint --fix to automatically fix safe errors.
M2
Method 2 — Trigger a Warning in the Editor

Since you already have this rule in your config:

.eslintrc.json 
"@typescript-eslint/no-unused-vars": "warn"

Open any .ts file (e.g. app.component.ts) and add this inside a method:

app.component.ts 
ngOnInit() {
  const unusedVariable = 'hello';  // ESLint should warn here
}

What to look for:

〰️ A yellow squiggly underline appears under unusedVariable
💬 Hovering over it shows: 'unusedVariable' is assigned a value but never used
M3
Method 3 — Check the Problems Panel

Press Ctrl+Shift+M to open the Problems panel at the bottom of VS Code. It shows all ESLint warnings and errors across your entire project in real time.

Column What it shows
Description The ESLint rule that was violated
File / Line Exact location of the issue
Severity Error (red) or Warning (yellow)
Troubleshooting
Problem Fix
No squiggles in editor Check ESLint Output (Ctrl+Shift+U), restart VS Code
ng lint command not found Run ng add @angular-eslint/schematics first
ESLint errors on HTML templates Add "html" to eslint.validate in settings.json
Quick Reference
bash 
# Install ESLint in Angular project
ng add @angular-eslint/schematics

# Run linter across whole project
ng lint

# Auto-fix safe errors
ng lint --fix

# Lint a specific file
npx eslint src/app/app.component.ts
