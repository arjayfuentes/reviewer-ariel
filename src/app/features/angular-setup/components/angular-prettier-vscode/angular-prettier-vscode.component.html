Prettier Setup Guide

Angular + VS Code — Code Formatting from Installation to Configuration

What is Prettier?

Prettier is a code formatter — it automatically rewrites your code to look consistent every time you save. It does not find bugs (that is ESLint's job), it just makes your code clean and uniform.

✗ Before Prettier 
const user={name:'John',
age:30,active:true}
function greet(
  name:string){
  return 'Hello '+name}
✓ After Prettier 
const user = {
  name: 'John',
  age: 30,
  active: true,
};
function greet(name: string) {
  return 'Hello ' + name;
}
ESLint vs Prettier
ESLint Prettier
Purpose Finds code quality issues Formats code appearance
Catches Unused vars, bad patterns Spacing, quotes, indentation
Output Errors & warnings Rewrites your code
Works on TypeScript logic TS, HTML, CSS, JSON
💡 They work together — ESLint catches bugs, Prettier handles style.
Installation
01
Install the VS Code Extension

Press Ctrl+Shift+X to open the Extensions panel. Search for Prettier - Code Formatter and install the one by Prettier.

Extension ID 
esbenp.prettier-vscode  (Prettier)
02
Install Prettier in Your Angular Project

Run this in your project terminal:

bash 
npm install --save-dev prettier
03
Create a .prettierrc Config File

Create a file called .prettierrc in your project root:

.prettierrc 
{
  "singleQuote": true,
  "semi": true,
  "tabWidth": 2,
  "printWidth": 100,
  "trailingComma": "es5"
}

What each option does:

Option Effect
singleQuote uses ' instead of "
semi adds semicolons at end of lines
tabWidth 2 spaces indentation (Angular default)
printWidth wraps lines longer than 100 characters
trailingComma adds trailing commas where valid in ES5
04
Update Your settings.json

Press Ctrl+Shift+P, type Open User Settings JSON, and press Enter. Your settings.json should look like this:

settings.json 
{
  "[html]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[typescript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[javascript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "editor.formatOnSave": true,
  "security.workspace.trust.untrustedFiles": "open",
  "javascript.updateImportsOnFileMove.enabled": "always",
  "typescript.updateImportsOnFileMove.enabled": "always",
  "explorer.confirmDelete": false,
  "git.autofetch": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": "explicit"
  },
  "eslint.validate": [
    "javascript",
    "typescript",
    "html"
  ]
}
💡 The key additions are [typescript] and [javascript] blocks — these tell VS Code to use Prettier for .ts and .js files, not just .html.
05
Avoid ESLint + Prettier Conflicts

Since ESLint also has some formatting rules, they can conflict with Prettier. Install this package to turn off ESLint rules that Prettier already handles:

bash 
npm install --save-dev eslint-config-prettier

Then add prettier as the last item in your eslint.config.js extends:

eslint.config.js 
extends: [
  eslint.configs.recommended,
  tseslint.configs.recommended,
  angular.configs.tsRecommended,
  "prettier"   // add this last
]
⚠️ prettier must always be last in the extends array so it overrides any conflicting formatting rules.
Testing Prettier Works
T1
Mess Up Formatting and Save

Open any .ts file and type something with bad formatting:

✗ Before save 
const x={a:1,b:2,c:3}
✓ After Ctrl+S 
const x = { a: 1, b: 2, c: 3 };
💡 If nothing happens, check that the file language is set to TypeScript (bottom-right of VS Code status bar).
T2
Format Manually

You can also format any file manually without saving:

Shift + Alt + F

Or run Prettier across your entire project from the terminal:

bash 
npx prettier --write .
