Using Maven

Build lifecycle, phases and running Spring Boot with Maven

⌨️
Maven Wrapper Syntax
bash 
mvnw [options] [<goal(s)>] [<phase(s)>]
01
Clean Phase

The clean lifecycle removes all build files from the target folder.

mvnw phases
clean
default
site
bash 
./mvnw clean
💡 Removes all build files from the target folder.
02
Default Phase

The default lifecycle handles building and deploying your project.

Phase Description
compile compiles code into bytecode
test runs unit tests
package creates a jar or war file
verify runs checks & integration tests
bash 
./mvnw compile
💡 Creates build files in the target folder.
03
Maven Plugin — Run Spring Boot

Use the Spring Boot Maven plugin to run your application directly.

bash 
./mvnw spring-boot:run
04
Run with Config — Set Server Port

Override configuration like server port directly from the Maven command line. Two ways to do this:

Option 1 — Environment Variable (PowerShell)

powershell 
$env:SERVER_PORT=8383; ./mvnw spring-boot:run

Option 2 — Spring Boot Run Arguments

bash 
./mvnw spring-boot:run "-Dspring-boot.run.arguments=--server.port=8383"
💡 Both options override the server port without touching application.properties.
