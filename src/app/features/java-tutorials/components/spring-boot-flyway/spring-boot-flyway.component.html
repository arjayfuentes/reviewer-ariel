Using Flyway

Database version control with Flyway migrations

01
Add Flyway dependencies in pom.xml

Add the Flyway Spring Boot starter and PostgreSQL-specific Flyway dependency to your pom.xml.

pom.xml 
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-flyway</artifactId>
</dependency>

<dependency>
  <groupId>org.flywaydb</groupId>
  <artifactId>flyway-database-postgresql</artifactId>
</dependency>
02
Reload Maven Projects

After adding the dependencies, reload Maven to download them.

Maven → Reload All Maven Projects
03
Create a migration SQL file

Under resources/db/migration create a SQL file following Flyway's naming convention:

📁 resources/
└── 📁 db/
    └── 📁 migration/
        └── V1__create_item_table.sql new
V1__create_item_table.sql 
CREATE TABLE item (
  id           UUID PRIMARY KEY,
  name         VARCHAR(255) NOT NULL,
  description  TEXT,
  price        NUMERIC(19, 2),
  image_url    VARCHAR(500),
  category     VARCHAR(100)
);

-- Optional: Add some starting data
INSERT INTO item (id, name, description, price, category)
VALUES (gen_random_uuid(), 'Milk', 'Fresh whole milk', 3.50, 'Dairy');
💡 Flyway naming convention: V{version}__{description}.sql — two underscores between version and description.
04
Add Flyway config in application.properties

Add the following properties to your application.properties file to configure Flyway and prevent Hibernate from interfering.

application.properties 
# This tells Hibernate not to interfere with Flyway
spring.jpa.hibernate.ddl-auto=validate

# Optional: specifically tell Flyway where to look
spring.flyway.locations=classpath:db/migration
spring.flyway.enabled=true
