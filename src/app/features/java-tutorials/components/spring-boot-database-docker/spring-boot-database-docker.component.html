PostgreSQL using Docker Compose

Set up a PostgreSQL database locally with Docker

🐳 Docker Guide
01
Create a docker-compose.yml file

Create a docker-compose.yml file in the root of your project. The PostgreSQL port to use is 5436.

02
Add the Docker Compose configuration

Paste the following configuration into your docker-compose.yml file.

docker-compose.yml 
services:
  db:
    image: postgres:latest
    container_name: reserba-postgres
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
      POSTGRES_DB: reserba
    ports:
      - "5436:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
03
Turn on Docker Desktop

Make sure Docker Desktop is running before executing any Docker commands. You should see the Docker icon active in your system tray.

💡 If Docker Desktop is not installed, download it from docker.com and install it first.
04
Run Docker Compose

In your terminal, navigate to the folder where docker-compose.yml is located and run the command below.

bash 
docker-compose up -d
💡 The -d flag runs the container in detached mode (in the background). PostgreSQL will be available on localhost:5436.
